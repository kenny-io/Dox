# Status Codes

API responses share the same error schema and status codes regardless of channel.

## Success

| Code | Meaning      |
| ---- | ------------ |
| 200  | OK           |
| 201  | Accepted     |
| 202  | Queued       |

## Retryable errors

| Code | Description              | Notes                            |
| ---- | ------------------------ | -------------------------------- |
| 408  | Request Timeout          | Retry with backoff.              |
| 429  | Rate limited             | Respect `Retry-After` header.    |
| 503  | Upstream unavailable     | Retries allowed for 24 hours.    |

```json
{
  "error": {
    "code": "rate_limited",
    "message": "Workspace exceeded the per-minute quota.",
    "retryAfter": 15
  }
}
```

<Note type="danger" title="Non-retryable codes">
400-level errors other than 408 and 429 represent malformed requests. Fix the payload before resubmitting.
</Note>
